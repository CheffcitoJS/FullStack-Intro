¿Qué es Git?

Git es un sistema de control de versiones distribuido que permite rastrear cambios en el código y colaborar en proyectos. Es una herramienta esencial para desarrolladores, ya que facilita el trabajo en equipo, el manejo de diferentes versiones del código y la resolución de conflictos de manera eficiente.

Características Principales

Distribuido

Cada desarrollador tiene una copia completa del repositorio, incluyendo su historial. Esto significa que no es necesario estar conectado a un servidor central para trabajar.

Velocidad

Git está diseñado para realizar operaciones como commits, merges y comparaciones de cambios de manera rápida y eficiente.

Seguridad

Garantiza la integridad del código mediante funciones hash y protege el historial de modificaciones contra alteraciones no autorizadas.

Flexible

Se adapta a distintos flujos de trabajo, como el modelo centralizado, descentralizado o basado en ramas.

¿Cómo Funciona Git?

Repositorio

Un repositorio es un contenedor para el código, su historial y las configuraciones relacionadas. Puede ser local o remoto, y es el núcleo del trabajo con Git.

Commit

Un commit es una instantánea de los cambios realizados en el código. Cada commit tiene un identificador único y un mensaje descriptivo para documentar los cambios.

Branch

Las ramas (branches) permiten trabajar en nuevas funcionalidades o correcciones sin afectar la rama principal. Esto facilita el desarrollo paralelo.

Merge

La operación de merge combina los cambios de una rama en otra, resolviendo conflictos si es necesario y consolidando el trabajo del equipo.

Comandos Más Utilizados

Configuración Inicial

Estos comandos se utilizan para configurar el nombre y correo electrónico asociados con los commits en tu máquina.

git config --global user.name "Tu Nombre"
git config --global user.email "tuemail@example.com"
git config --list  # Verifica la configuración

Flujo Básico

Comandos esenciales para empezar a trabajar con Git, desde inicializar un repositorio hasta registrar y compartir cambios en el código.

git init  # Inicializa un nuevo repositorio local

git clone   # Clona un repositorio remoto a local

git add   # Agrega archivos al área de preparación (staging area)

git commit -m "Mensaje del commit"  # Registra los cambios con un mensaje descriptivo

git status  # Muestra el estado de los archivos en el repositorio

git log  # Muestra el historial de commits

git push  # Envía los cambios al repositorio remoto

git pull  # Descarga y combina cambios del repositorio remoto

Trabajo con Ramas

Los siguientes comandos permiten trabajar con ramas para gestionar diferentes líneas de desarrollo.

git branch   # Crea una nueva rama

git checkout   # Cambia a una rama existente

git checkout -b   # Crea y cambia a una nueva rama

git merge   # Fusiona una rama en la actual

git branch  # Lista todas las ramas

Otros Comandos Útiles

Comandos adicionales para resolver problemas y manejar el historial.

git diff  # Muestra los cambios entre el repositorio y el área de preparación

git reset   # Quita un archivo del área de preparación

git checkout --   # Descarta cambios no confirmados en un archivo

git branch -d   # Elimina una rama

git clone    # Clona un repositorio remoto en una ubicación específica
